Problem Definition & Constraints

  • Audience: Local communities, donors, volunteers, schools
  • Summary: Need a site to present projects, collect signups and enquiries
  • Constraints: Low budget, 4-week timeline, volunteer content contributors
  • Interactivity: Contact form, volunteer signup, image gallery, newsletter.
  • Complexity: ~6–8 pages (Home, About, Projects, Resources, Get Involved, Contact)
Expected outcomes

Clear communication, increased volunteer signups, easier donor contact

Contact Form — Prototype & Pseudocode

Client-side validation with progressive enhancement. In production, form submission should POST to secure backend and use CAPTCHA/rate-limiting.

Click to view pseudocode and flow 
PSEUDOCODE: Contact Form Validation & Submission (Client-side)

1. User fills: name, email, subject, message.
2. On submit:
   - Prevent default form POST.
   - Trim inputs.
   - If name empty -> show error near name field.
   - If email not valid (regex) -> show error.
   - If message too short (< 10 chars) -> show error.
   - If all pass -> disable submit button, show "Sending..." and send using fetch() to /api/contact (POST JSON).
   - On success -> show success message, clear form.
   - On failure -> show friendly error, re-enable button.

FLOW (steps):
[User] -> [Fill form] -> [Client-side validation] -> (pass) -> [AJAX POST] -> [Server response] -> [Show success/failure]

TESTS:
- Empty fields -> expect explicit errors.
- Invalid email -> expect invalid message.
- Large message (10k chars) -> expect length limit enforced.
- Network failure simulation -> show failure message and retry option.
We aim to reply within 5 business days.

Feedback & Testing Plan (summary)

Plan to collect feedback from 8–12 users (classmates / teachers / local volunteers). Focus on clarity, mobile layout, and form flow.

Feedback highlights (example)

  • Make CTA more visible on mobile — moved volunteer CTA into sticky footer on mobile
  • Shorten hero text — changed to a single-line mission statement
  • Ensure high-contrast buttons — increased contrast of primary CTA

Testing checklist

  1. Links: all internal links navigate correctly (test pages)
  2. Forms: validation errors, success messages, server error handling
  3. Compatibility: Chrome, Firefox, Edge, Safari; mobile viewport checks
  4. Accessibility: alt text, keyboard navigation, color contrast
  5. Performance: images optimized, lazy-loading images, gzip enabled on server
